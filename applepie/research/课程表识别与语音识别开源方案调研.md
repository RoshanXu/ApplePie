> 调研日期：2026-06-30
> 调研目标：寻找免费/开源方案，实现（1）拍照/PDF 课程表识别与结构化提取，（2）中文语音转文字

## 第一部分：课程表（照片/PDF）识别

### 一、场景分析

课程表的核心特征是**规则表格 + 合并单元格**（午休、连堂课等跨行/跨列）。技术链路：

```
照片/PDF → 预处理(纠偏/增强) → 表格区域检测 → 表格结构识别 → OCR文字识别 → 结构化输出(JSON/Excel)
```

### 二、开源方案对比

#### 🥇 PaddleOCR / PP-Structure（百度）— 中文课程表首选

| 项目 | 详情 |
|------|------|
| **GitHub** | `PaddlePaddle/PaddleOCR` |
| **许可证** | **Apache 2.0**（免费商用） |
| **Star** | 46k+ |
| **语言** | 80+ 种（中文支持最优） |
| **核心能力** | PP-Structure 专攻表格结构识别，支持合并单元格、跨行跨列、无边框表格 |
| **输出格式** | Excel / JSON / CSV / HTML |
| **部署** | 轻量部署，支持 CPU/GPU/树莓派 |
| **安装** | `pip install paddlepaddle paddleocr` |

**课程表适用性**：⭐⭐⭐⭐⭐
- 中文识别精度业界最高
- 合并单元格还原能力最强
- 可端侧部署，不依赖云 API
- 有完整的 Pipeline：布局检测 → 表格识别 → 文字提取 → 结构还原

---

#### 🥈 Surya OCR — 多语言布局解析新星

| 项目 | 详情 |
|------|------|
| **许可证** | 开源免费 |
| **核心能力** | 90+ 语言、图神经网络建模列关系、复杂排版（多栏/图文混排） |
| **表格精度** | 宣称 98%+ |
| **输出格式** | JSON / Excel / CSV |
| **部署** | 需 PyTorch，GPU 推荐 |

**课程表适用性**：⭐⭐⭐⭐
- 排版复杂时（如手写课表+贴纸装饰）有优势
- 中文精度略逊 PaddleOCR

#### 🥉 Nanonets-OCR-s — 端到端 VLM 方案

| 项目 | 详情 |
|------|------|
| **基座** | Qwen2.5-VL-3B |
| **许可证** | 开源 |
| **核心能力** | 端到端图像解析，直接输出 Markdown（含 HTML 表格），支持复杂表格、水印、公式、签名 |
| **部署** | 需 GPU（3B 模型） |

**课程表适用性**：⭐⭐⭐⭐
- 最省事：一张图进去，Markdown 表格出来，无需拼装多个模型
- 适合拍照课表（背景噪声多、光照不均、手写标注）
- 缺点：3B 模型推理较慢，手机端部署困难

### 五、课程表识别推荐技术组合

#### 方案 A：纯本地/免费/高精度（✅ 强烈推荐）

```
PaddleOCR/PP-Structure → 表格结构 JSON → 后处理规则 → 课程表 JSON
```

- ✅ 免费商用
- ✅ 无需 GPU
- ✅ 中文最优
- ✅ 可部署到服务器或端侧

#### 方案 B：VLM 端到端（省事但较重）

```
Nanonets-OCR-s / Qwen2.5-VL → Markdown 表格 → 解析为 JSON
```

- ✅ 开发量最小
- ❌ 需 GPU，推理较慢

#### 方案 C：PDF 优先路径

```
Camelot（提取表格）→ DataFrame → 后处理 → 课程表 JSON
（若为扫描件则先用 PaddleOCR 做 OCR 再用 Camelot）
```

#### 方案 D：多模型混合 Pipe（最高精度）

```
YOLOv8（表格区域检测）→ PaddleOCR（文字识别 + 结构识别）→ LLM（纠错 + 结构化）
```

## 第二部分：中文语音识别 (ASR)

### 一、核心开源模型一览（2025-2026）

| 模型 | 开发者 | 参数 | 许可证 | 中文 CER | 亮点 |
|------|--------|------|--------|---------|------|

| **Fun-ASR-Nano** | 通义实验室 | 800M | **Apache 2.0** | AISHELL-1 ~1.6% | 生态最成熟，FunASR 工具链 |

### 二、精度对比（CER% — 越低越好）

| AISHELL-2 | **2.51** | 2.71 | 2.38 | 4.96 |

### 三、端侧/移动端部署方案

#### sherpa-onnx（⭐ 10.8k — 端侧部署首选）

| 项目 | 详情 |
|------|------|
| **GitHub** | `k2-fsa/sherpa-onnx` |

| 项目 | 详情 |

| **平台** | Mac / iOS / Android / Linux / Web |

| 组件 | 功能 |

### 四、选型建议

| 场景 | 推荐方案 | 理由 |
|------|---------|------|
| **追求最高精度** | FireRedASR v2 | CER 最低，中文公开基准 SOTA |
| **方言多/国际化** | Qwen3-ASR | 52 语言 + 22 中文方言，Apache 2.0 |
| **移动端 App 集成** | **SenseVoice-Small + sherpa-onnx** | 模型小(234M)，推理极快，支持 iOS/Android |
| **最简单上手** | `pip install funasr` + Paraformer | 一行代码完成识别 |
| **Web/服务端部署** | Qwen3-ASR 或 Fun-ASR-Nano | 精度高，部署灵活 |
| **需要情感/情绪识别** | SenseVoice-Small | 独家支持 |
| **Whisper 生态已有投资** | whisper.cpp + fine-tuned 中文 Whisper | 仍需确认中文精度 |

### 五、快速上手指南

#### 方案 1：Python 服务端（推荐新手）

```python
# FunASR — 最简单
from funasr import AutoModel

model = AutoModel(model="iic/SenseVoiceSmall")
result = model.generate(input="audio.wav")
print(result[0]["text"])
```

```python
# Qwen3-ASR — 精度更高
from qwen_asr import Qwen3ASRModel

model = Qwen3ASRModel.from_pretrained("Qwen/Qwen3-ASR-1.7B")
result = model.transcribe("audio.wav")
```

#### 方案 2：移动端（iOS/Android）

```
集成 sherpa-onnx SDK → 加载 SenseVoice-Small ONNX 模型 → 本地离线推理
```

- 无需网络
- 延迟 < 100ms
- 模型大小：Paraformer ~93MB, SenseVoice-Small ~90MB

## 第四部分：总结与建议

### 课程表识别

| 维度 | 推荐 | 替代 |
|------|------|------|
| 通用图片/拍照课表 | **PaddleOCR + PP-Structure** | Surya OCR |
| PDF 课表（非扫描） | **Camelot** | pdfplumber + Marker |
| 开发速度优先 | **Nanonets-OCR-s**（VLM 端到端） | PaddleOCR |
| 最高精度 | YOLOv8 + PaddleOCR + LLM 纠错 | — |

### 语音识别

| 维度 | 推荐 | 替代 |
|------|------|------|
| 移动端 App | **SenseVoice-Small + sherpa-onnx** | Fun-ASR-Nano |
| 服务端/高精度 | **FireRedASR v2** 或 **Qwen3-ASR** | Fun-ASR-Nano |
| 方言需求 | **Qwen3-ASR**（22 种方言） | FireRedASR |
| 最简单上手 | **FunASR + Paraformer** | — |

### 关键结论

1. **所有推荐方案均为免费开源 + 可商用**（Apache 2.0 / MIT），无授权风险
2. 中文场景优先选国产模型：PaddleOCR（OCR）+ 国产 ASR（语音），精度远超国际方案
3. 端侧部署可行：PaddleOCR 支持轻量部署；sherpa-onnx 让 ASR 在手机上离线运行
4. 2025-2026 年中文 ASR 爆发：FireRedASR、Qwen3-ASR、Fun-ASR-Nano 三家竞争，精度均已远超 Whisper
