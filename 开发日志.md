## 二、架构决策记录（ADR）

### ADR-001：选择 infiplot 作为 AI 游戏引擎

- **决策：** MVP 阶段直接复用 [infiplot](https://github.com/zonghaoyuan/infiplot) 的互动叙事引擎，不改核心代码，只替换输入层（`worldSetting`）。
- **理由：**
  - infiplot 已实现完整的"AI 编剧→角色设计→分镜→生图→TTS"多智能体流水线
  - Session 模型携带 worldSetting（自由文本），天然支持注入教育上下文
  - ContextSegment 注册表模式允许追加自定义上下文而不侵入引擎
  - Next.js 16 + TypeScript，技术栈成熟
- **替代方案：** 自研引擎（MVP 周期过长，预估多 6-8 周）

### ADR-002：核心功能永久免费 + 增值服务变现

- **决策：** 时间管理（含 AI 导入）、AI 游戏、基础能力画像永久免费。实践活动报名、专业课程、跳蚤市场变现。
- **理由：**
  - 时间管理是高频入口，免费最大化用户基数
  - AI 游戏是数据引擎，免费才能积累足够训练数据
  - 变现发生在"发现需求→满足需求"的自然延伸节点，不伤体验
- **替代方案：** Freemium（会员订阅解锁 AI 游戏次数）—— 被否决，会阻碍数据积累飞轮

### ADR-003：MVP 只做 5 个模块

- **决策：** MVP 范围 = AI 导入日程 + 五维健康度 + 1 套 AI 游戏 + 基础能力画像 + 家长基础周报
- **砍掉：** 实践推荐、互动社区、跳蚤市场、家长反思区、多套游戏主题
- **理由：** 先验证"时间管理→AI 游戏→能力画像"核心闭环

### ADR-004：五维时间模型

- **决策：** 使用学习/休息/运动/社交/探索五个维度，而非传统的"学习/生活"二分。
- **理由：** "均衡成长"是产品核心理念——鼓励孩子不只学习好，而是活得好。这也是与竞品（作业帮、小猿搜题等）的最大差异点。

### ADR-005：AI 游戏作为发现引擎，实践推荐作为延伸出口

- **决策：** AI 游戏和实践推荐不是互斥的二选一，而是递进关系：游戏发现兴趣/薄弱点 → 推荐匹配的活动。
- **理由：** 解决了"个性化活动推荐"的冷启动问题——游戏数据提供推荐信号。

## 三、技术调研记录

### 3.1 infiplot 引擎分析

| 维度 | 评估 |
|------|------|
| **架构** | 无状态服务端，客户端携带完整 Session。多智能体流水线：Writer → CharacterDesigner ∥ Cinematographer → Painter → Assembly |
| **核心技术** | Next.js 16 + React 19 + TypeScript，AI 模型通过 `openai_compatible` 协议接入 |
| **图像** | Runware / OpenAI 兼容（FLUX），支持角色肖像一致性（referenceImages） |
| **TTS** | 小米 MiMo / 步非烟（StepFun），可选 |
| **Prompt 系统** | ContextSegment 注册表区分 stable/dynamic 区，支持 prompt 前缀缓存 |
| **可复用度** | 🟢 高 — 引擎核心零改动，只替换 worldSetting 内容 + 新增 API 路由封装 |

### 3.2 关键技术风险

| 风险 | 状态 | 应对 |
|------|------|------|
| AI 图片生成成本 | 🟡 关注 | MVP 可用 `MOCK_IMAGE=true` 跳过生图，先用纯文本+占位图跑通 |
| 大模型 API 延迟 | 🟡 关注 | infiplot 已实现流式 SSE + 并行管线，Writer 和图像并行 |
| 课表 OCR 准确率 | 🔴 待验证 | 需测试不同光照/角度/手写课表的识别效果，必要时刻专用 OCR 模型微调 |
| 语音解析鲁棒性 | 🔴 待验证 | 中文口语→结构化日程的鲁棒性需实测 |
