## 二、架构决策记录（ADR）

### ADR-001：选择 infiplot 作为 AI 游戏引擎

- **决策：** MVP 阶段直接复用 [infiplot](https://github.com/zonghaoyuan/infiplot) 的互动叙事引擎，不改核心代码，只替换输入层（`worldSetting`）。
- **理由：** infiplot 已实现完整的"AI 编剧→角色设计→分镜→生图→TTS"多智能体流水线

### ADR-002：核心功能永久免费 + 增值服务变现

- **决策：** 时间管理、AI 游戏、基础能力画像永久免费。实践活动、专业课程、跳蚤市场变现。

### ADR-003：MVP 只做 5 个模块

- **决策：** MVP = AI 导入日程 + 五维健康度 + 1 套 AI 游戏 + 基础能力画像 + 家长基础周报

### ADR-004：五维时间模型

- **决策：** 学习/休息/运动/社交/探索五个维度，"均衡成长"为核心理念

### ADR-005：AI 游戏作为发现引擎

- **决策：** 游戏发现兴趣/薄弱点 → 推荐匹配活动

### ADR-006：知识点剧集化游戏

- **决策：** 每集约 5 分钟围绕一个知识点，结构：引入→应用→陷阱/拓展→总结+钩子
- **理由：** 时间太长兴趣锐减，短剧集保持专注，按知识点组织便于能力分析

### ADR-007：移除 Supabase，使用自定义 JWT + PostgreSQL

- **决策：** 移除 Supabase Auth/RLS，用 `jose` + `bcryptjs` 实现自定义 JWT 认证，用户表直接存 passwordHash
- **理由：** Supabase 本地部署需要 12 个 Docker 容器，维护成本高。App 只用到了 Auth（6 个文件），数据访问全走 Prisma，不需要 PostgREST/RLS。替换后 Docker 仅需 2 个容器

### ADR-008：Docker 镜像离线分发

- **决策：** `docker save` 打包 app + postgres 两个镜像为一个 tar 文件（~400MB），目标机器 `docker load` 后直接 `docker compose up -d`
- **理由：** 目标机器不需要拉取镜像、不需要 Node.js/pnpm、不需要源码，仅需 Docker 运行时

## 五、数据库

- **PostgreSQL 16** — Docker 容器，端口 5432
- **12 张表** — users（含 passwordHash/phone unique）, student_profiles, schedules（含 sub_category/subject/completed）, knowledge_points, game_sessions 等
- **Auth**：自定义 JWT（jose + bcryptjs），httpOnly cookie，7 天有效期
- **Schedule 模型扩展**：`subCategory`（二级分类）、`subject`（三级科目）、`completed`（完成状态）
- **81 个知识点**已种子（七年级-九年级数学+物理，人教版）
