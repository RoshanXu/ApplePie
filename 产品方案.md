# ApplePie 产品方案

> **AppleU，成长可以快乐，把童年还给孩子。**

## 一、产品概述

### 1.1 产品定位

ApplePie 是一款面向**中学生及其家长**的 AI 驱动学习成长应用。以**时间管理为入口、AI 互动游戏为引擎、线下实践为延伸**，帮助学生从"被动刷题"转向"主动探索"，同时引导家长从"焦虑监工"转向"陪伴成长"。

**核心理念：** 帮每个孩子找到自己的学习节奏和兴趣方向，而不是把所有人都赶上同一座独木桥。

**定价策略：** 应用本身**免费下载、免费使用**。核心功能（时间管理、AI 游戏、基础画像）永不收费。通过实践活动、专业课程、跳蚤市场等增值服务变现。

### 1.2 目标用户

| 用户层 | 描述 | 核心诉求 |
|--------|------|---------|
| 核心用户（学生） | 12-18 岁中学生，作业压力大，对填鸭式学习疲惫，有潜在兴趣但未被发掘 | "我想学得轻松一点，也想找到自己真正喜欢的" |
| 付费用户（家长） | 35-50 岁，焦虑孩子的学业和未来，愿意为"有效果"的教育方式付费 | "我想帮孩子，但不知道怎么做才对" |

### 1.3 核心价值主张

```
传统模式：上课 → 刷题 → 考试 → 焦虑 → 报更多课 → 刷更多题
ApplePie：时间管理 → AI游戏（发现薄弱+兴趣） → 精准实践 → 能力成长 → 家长看见
```

三个差异化：
1. **不是教辅工具，是成长系统** — 管时间、管兴趣、管实践，不只管分数
2. **AI 不是答题机，是发现引擎** — 通过游戏了解孩子，而非通过考试评价孩子
3. **家长不是旁观者，是参与者** — 有专属的成长板块，不是"监控面板"

## 二、产品架构

### 2.1 功能模块全景

```
ApplePie
├── 🏠 首页仪表盘         — 今日概览、时间进度、AI 游戏入口
├── 📅 时间管理           — 五维时间规划（学习/休息/运动/社交/探索）
├── 🎮 AI 互动游戏        — 基于知识点+错题+兴趣实时生成的图文游戏
├── 🔍 实践推荐           — 基于游戏表现推荐的线下/线上实践活动
├── 💬 互动交流           — AI 助手 + 同伴分享
├── 👨‍👩‍👧 家长空间         — 家长反思、成长记录、闪光点日志
└── 👤 账号管理           — 学生/家长双板块，角色切换
```

### 2.2 核心关系

```
时间管理（高频入口，黏性来源）
    │
    │  规划每日时间，记录学习数据
    ▼
AI 互动游戏（核心引擎，差异化）
    │
    │  发现：薄弱知识点、兴趣方向、潜在特长
    │  输出：知识点掌握画像 + 兴趣特长信号
    ▼
实践推荐（变现出口）
    │
    ├── 附近实践活动（线下，收费/合作分成）
    ├── 专业进阶课程（线上/线下，内容付费）
    └── 跳蚤市场（二手书/笔记/手工作品，抽佣）

家长空间（留存 + 付费转化）
    │
    │  看到孩子的成长轨迹 → 认可价值 → 愿意为实践/课程付费
    ▼
互动交流（社区氛围，口碑传播）
```

## 三、功能详细设计

### 3.1 时间管理（入口 · 免费 · 高频 · 第一吸引力）

**定位：** 应用的第一入口，用户下载后最先接触、最频繁使用的功能。核心体验是 **"上传即生成"**——用户拍照/语音/文档丢进去，AI 自动生成完整的时间计划表和五维健康分析。

#### 3.1.1 AI 智能录入（杀手功能）

用户不需要手动一条条添加日程。支持三种输入方式，可组合使用：

**输入方式一：图片识别**
| 输入 | AI 提取 |
|------|---------|
| 📸 课程表照片 | 周一至周五课程安排、节次时间、科目名称、任课老师 |
| 📸 通知单/家长群截图 | 考试日期、活动通知、作业截止日 |
| 📸 学生手写计划 | 手写的学习/活动安排 |

**输入方式二：语音描述**
```
用户按住说：
"周二下午5点上围棋课，到6点半，在少年宫。
周五晚上7点到8点半有篮球班。
这周六上午要去参加数学竞赛，在新华中学。
每天晚上9点到10点我要留出来写作业。"

→ AI 解析为结构化日程，一键添加到日历
```

**输入方式三：文档导入**
- 学校发的 Word/PDF 校历、教学计划
- 家长群转发的日程表

#### 3.1.2 一键生成日程

```
用户操作流程：

1. 拍一张课表照片 ────────────────┐
2. 语音说："周二17点围棋课，       │
   周五19点篮球班"                 │
3. （可选）再上传考试安排表         ├──→ AI 聚合解析
                                    │
                                    ▼
                          ┌─────────────────┐
                          │  AI 生成预览页    │
                          │                  │
                          │ ✓ 周一            │
                          │  08:00-08:45 语文 │
                          │  09:00-09:45 数学 │
                          │  ...             │
                          │  17:00-18:30 围棋 │ ← 来自语音
                          │                  │
                          │ ✓ 周五            │
                          │  ...             │
                          │  19:00-20:30 篮球 │ ← 来自语音
                          │                  │
                          │ ✓ 周六            │
                          │  09:00-11:00     │ ← 来自图片
                          │  数学竞赛(新华中学)│
                          │                  │
                          │ [一键确认添加]    │
                          │ [编辑调整]        │
                          └─────────────────┘
```

**AI 解析能力：**
- 时间表达式归一化："下周二""每周三""明天下午""5点" → 标准日期时间
- 重复规则自动识别：课表自动设为每周重复，考试/活动设为单次
- 地点提取：从描述中提取"少年宫""新华中学"并关联地图
- 冲突检测：同一时段两个安排 → 提醒用户确认

#### 3.1.3 五维时间模型 & 智能提醒

**五维自动分类：** AI 导入日程时，自动将每条事项归入五维之一：

| 维度 | 含义 | 示例 |
|------|------|------|
| 📚 学习 | 课内学习、作业、考试 | 语文课、数学作业、期中考试 |
| 😴 休息 | 睡眠、午休、放松 | 午休、晚上睡觉（自动根据结束时间推算） |
| 🏃 运动 | 体育锻炼 | 篮球班、晨跑 |
| 💬 社交 | 朋友、家庭互动 | （AI 游戏后自动识别，或手动标记） |
| 🔍 探索 | 兴趣发展、实践 | 围棋课、AI游戏、手工 |

**智能均衡分析（五维健康度）：**

导入日程后，AI 自动计算本周五维时间占比，并与推荐区间对比：

```
本周时间分配：

📚 学习  ████████████████████████  58%  ⚠️ 偏高（推荐 40-50%）
😴 休息  ████████                  22%  ✓ 正常（推荐 20-30%）
🏃 运动  ████                       8%  ⚠️ 偏低（推荐 10-15%）
💬 社交  ██                          4%  ❌ 严重不足（推荐 8-12%）
🔍 探索  ██                          8%  ⚠️ 偏低（推荐 10-15%）

💡 本周建议：
  · 学习时间偏长，可以考虑把部分刷题换成 AI 游戏
  · 社交时间太少！周末可以约朋友一起户外活动
  · 篮球班只有一次，建议增加一次自主运动
```

**提醒机制：**
- **维度失衡预警**：某维度连续两周严重偏离 → 推送提醒家长和孩子关注
- **日记账**：每晚 21:00 推送"今天的时间都去哪了"简短复盘
- **正向鼓励**：当孩子主动增加运动/探索时间 → 给成就徽章，家长端同步显示

#### 3.1.4 日历视图与日常操作

- **周视图/日视图**：五维颜色标签，一眼看清时间分配
- **拖拽调整**：长按事项拖动调整时间
- **完成打卡**：勾选完成，未完成一键顺延到明天
- **快速模板**：手动添加时提供模板（"放学后""周末晨间"）
- **跨端同步**：学生端调整 → 家长端实时可见（成长视角，非监控视角）

#### 3.1.5 设计原则

- **3 秒完成录入**是上限，不是目标。AI 导入把录入成本从"分钟级"降到"秒级"
- 鼓励平衡而非"学习越多越好"——五维健康度分析是本功能的灵魂
- 家长端看到的是时间分配和完成率，不显示具体考试分数排名

### 3.2 AI 互动游戏（引擎 · 核心差异化）

**定位：** 应用的心脏。不是"玩物丧志"的游戏，而是**以故事为壳、以知识点为核**的互动学习体验。孩子在玩的过程中暴露薄弱点、展现兴趣倾向。

**游戏机制：**

```
主线故事（预设框架，按学科/年龄段设计多套）
    │
    │  例如：星际探险（物理/数学）、古文明解密（历史/语文）、生态守护（生物/地理）
    │
    ├── 场景 A → 遇到挑战（AI实时生成）
    │   ├── 需要运用"二次函数"知识 → 检测掌握程度
    │   ├── 需要读懂一段文言文 → 检测理解能力
    │   └── 需要设计实验方案 → 检测思维偏好
    │
    ├── 场景 B → 分支选择（AI实时生成）
    │   ├── 选择"动手制作"路径 → 动手能力信号 +1
    │   ├── 选择"逻辑推理"路径 → 逻辑思维信号 +1
    │   └── 选择"团队协作"路径 → 社交能力信号 +1
    │
    └── 场景 C → 复盘总结（AI实时生成）
        ├── 你做对了什么（知识点已掌握）
        ├── 哪里可以更好（薄弱环节）
        └── 你展现的特质（兴趣/特长信号）
```

**技术实现参考：**
- 参考 [infiplot](https://github.com/zonghaoyuan/infiplot) 的 AI 互动叙事引擎
- 主线框架：预设故事主题、场景结构、角色设定（设计师/策划配置）
- 互动内容：AI 实时生成对话、题目、分支选项（接入大模型 API）
- 输出数据：知识点正确率、选择偏好、解题路径、耗时

**与学习的关联：**
- 游戏中的知识点来自：当前学校进度 + 历史错题 + 薄弱环节
- 游戏难度自动调节：根据孩子水平动态调整，保持"跳一跳够得着"
- 游戏时长控制在 15-25 分钟，一天 1-2 局

**关键设计约束：**
- 以图文互动为主（文本 + 插画/漫画风），不用 3D 或复杂动画（降低开发成本和技术门槛）
- 所有 AI 生成内容需经过内容安全过滤
- 游戏数据归入学生能力画像，但原始数据不直接展示给家长（避免"又一个考试"的感觉）

### 3.3 实践推荐（变现出口）

**定位：** AI 游戏的延伸。游戏发现了孩子的兴趣和薄弱点，实践推荐给出"线下真实世界"的对应活动。

**推荐逻辑：**

```
AI 游戏画像
    │
    ├── 薄弱知识点：二次函数、文言文虚词
    ├── 兴趣倾向：动手制作（信号强度 0.8）、逻辑推理（0.6）
    ├── 特长信号：空间想象力突出
    └── 当前课程：初二数学（函数章节）、初二语文（唐宋散文）
            │
            ▼
    推荐引擎
    ├── 规则 1：与薄弱知识点相关 → 巩固型活动
    ├── 规则 2：与兴趣倾向匹配 → 发展型活动
    ├── 规则 3：与当前课程同步 → 辅助型活动
    ├── 规则 4：地理位置可达 → 线下的优先
    └── 规则 5：价格在家庭承受范围
            │
            ▼
    输出活动列表（排序：匹配度 × 距离 × 评价）
```

**活动类型：**

| 类型 | 示例 | 变现方式 |
|------|------|---------|
| 🏛️ 场馆活动 | 科技馆物理实验课、博物馆历史探秘 | 平台合作抽佣 15-20% |
| 🎨 工作坊 | 编程入门工作坊、木工手作体验 | 平台合作抽佣或自营 |
| 📖 研学活动 | 周末古建筑研学、植物园生态考察 | 平台合作抽佣 |
| 🎓 专业课程 | 物理竞赛进阶、创意写作课 | 内容方入驻分成或自研 |
| 🔄 跳蚤市场 | 二手教辅/笔记/手工作品交易 | 交易抽佣 5-10% |

**活动详情页包含：**
- 活动介绍、时间、地点、适合年龄段
- **关联知识点标签**（如"涉及：勾股定理、力学基础"）
- 往期参与者反馈
- 家长报名入口（付费环节）

---

### 3.4 互动交流（社区氛围）

**定位：** 活动后的分享和沉淀，增强社区归属感。不追求大而全的社交，聚焦"活动后复盘"。

**功能点：**
- **活动后 AI 引导复盘**：每次 AI 游戏或实践活动结束后，AI 发起几个问题引导学生记录感受
- **学习经验帖**：学生可以发布自己的学习方法、笔记分享（类似小红书的图文笔记）
- **AI 学习助手**：随时可对话，答疑、讲解知识点、学习方法建议
- **同伴互评**：对公开分享的内容，其他学生可以点赞、鼓励性评论

**设计原则：**
- 弱化"比较"和"排名"，强调"分享"和"鼓励"
- AI 助手承担答疑角色，降低学生提问的心理门槛
- 所有公开内容需审核，保护未成年隐私

---

### 3.5 家长空间（留存 · 付费转化）

**定位：** 家长不是在"监控"孩子，而是在"发现自己"。家长的反思和成长本身就是给孩子最好的教育。

**功能点：**

| 功能 | 说明 |
|------|------|
| **孩子成长周报** | 不是成绩单，而是：时间分配变化、AI游戏中展现的闪光点、完成的实践活动、兴趣倾向变化 |
| **闪光点日志** | 家长记录观察到的孩子进步/闪光点（哪怕很小），AI 帮助整理和回顾 |
| **家长反思区** | 引导问题："这周我为孩子做了什么？哪些是孩子真正需要的？我自己收获了什么？" |
| **教育知识库** | 精选文章/课程：青春期心理学、学习方法论、亲子沟通技巧（免费引流+付费深度内容） |
| **家庭计划** | 家长和孩子共同设定家庭活动计划（如"每月一次博物馆"），双向参与 |

**设计原则：**
- 家长端的核心设计理念是 **"看见成长，而非看见分数"**
- 周报不展示具体的考试分数、排名，而是展示时间投入、能力变化、兴趣信号
- 家长反思区是私密的（仅自己可见），降低表达心理门槛

---

### 3.6 账号管理

**双向角色设计：**

```
一个账号 = 学生角色 + 家长角色（初始注册时选择）

学生端界面：
- 首页（今日计划 + AI游戏入口）
- 时间管理
- AI 游戏
- 实践推荐
- 互动社区

家长端界面：
- 家长首页（孩子成长概览 + 自己未完成的反思）
- 成长周报
- 闪光点日志
- 反思区
- 教育知识库
- 家庭计划

切换方式：底部 Tab "切换到家长/学生视角"
```

**设计原则：**
- 同一设备，一键切换角色视图
- 学生在家长端看到的内容是"成长视角"的，不是"监控视角"的
- 家长也可以自己注册独立账号，绑定孩子账号（适合家长用自己的手机）

## 四、商业模式

### 4.1 收入模型

**核心原则：应用本身不收费。时间管理、AI 游戏、基础画像 —— 永久免费，不限制使用次数。**

```
┌─────────────────────────────────────────┐
│  🆓 永远免费（引流 + 留存 + 数据积累）      │
│  时间管理（含AI导入）、AI 游戏（不限次）、   │
│  基础能力画像、AI 学习助手、基础社区        │
├─────────────────────────────────────────┤
│  💰 实践活动（核心变现）                    │
│  · 线下活动报名抽佣 15-20%                 │
│  · 自营精品活动（高毛利）                   │
│  · 机构入驻年费/保证金                      │
├─────────────────────────────────────────┤
│  💰 专业课程（内容变现）                    │
│  · 进阶课程销售分成 20-30%                 │
│  · 自研 AI 学习力课程                      │
│  · 专家辅导/1v1 咨询                       │
├─────────────────────────────────────────┤
│  💰 跳蚤市场（交易抽佣）                    │
│  · 二手教辅/笔记/手工作品 交易抽佣 5-10%    │
├─────────────────────────────────────────┤
│  💰 增值服务（可选）                        │
│  · 深度能力画像报告（一次性付费 ¥9.9）       │
│  · 教育知识库深度内容（单篇付费/打包）        │
└─────────────────────────────────────────┘
```

**关键差异：** 与"免费增值（freemium）"模式不同，ApplePie 不是"免费试用再收费"，而是核心体验**永不收费**。变现发生在用户获得价值之后的延伸场景——当 AI 发现孩子喜欢物理实验，推荐一个周末科学馆活动，家长为此付费。这是"需求发现 → 满足需求"的自然转化，不是"卡脖子"收费。

### 4.2 增长飞轮

```
免费 AI 导入课表/语音 → 一键生成日程（用户首次"哇"时刻）
    ↓
五维健康度提醒 → 每天打开看时间分配 → 高频使用、数据积累
    ↓
AI 游戏发现兴趣/薄弱点 → 推荐匹配实践活动 → 家长首次付费
    ↓
家长看到真实成长（闪光点日志） → 愿意分享/推荐 → 口碑传播
    ↓
更多用户 → 更多数据 → AI 更精准 → 推荐更匹配 → 更多付费转化
```

## 六、关键风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| AI 图片/语音识别准确率不足 | 用户首次体验差，流失 | 课表识别用专用 OCR 微调模型；语音解析失败时提供手动修正入口；识别结果始终可编辑 |
| AI 生成内容质量不稳定 | 游戏体验差，用户流失 | 预设内容兜底 + 人工审核模板 + 持续调优 prompt |
| 家长对"免费"可持续性存疑 | 影响信任和初期增长 | 公开透明说明商业模式（实践推荐/课程变现），强调"核心功能永不收费" |
| 线下活动供给不足 | 推荐为空，变现断流 | 前期聚焦一二线城市（活动供给足），逐步拓展；同时发展线上实践作为补充 |
| 未成年人隐私合规 | 法律风险 | 参考《个人信息保护法》《未成年人保护法》，最小化数据收集，家长可控 |
| 大模型 API 成本高 | 单位经济模型差 | 缓存常见场景模板、游戏内容批量预生成、混合使用便宜模型 |
